Exercise Overview: Conversational RAG with Memory (TDD Approach)
🎯 Goal
Build a locally hosted chatbot capable of Retrieval-Augmented Generation (RAG) that maintains conversation history. The system must answer questions based on provided documents while remembering previous turns to handle follow-up queries (e.g., "What is X?" followed by "How does it work?").

🧠 Learning Objectives
This exercise is designed to bridge the gap between theoretical LLM knowledge and production-grade engineering practices.

LangChain Mastery: Understand ConversationalRetrievalChain and ConversationBufferMemory.
Vector Search: Learn how to ingest, chunk, embed, and retrieve documents using a local Vector Database.
Test-Driven Development (TDD): Apply TDD principles to non-deterministic AI systems.
Containerization: Orchestrate microservices (LLM, Vector DB, App) using Docker Compose.
AI Testing: Learn strategies for testing LLM output (semantic assertions vs. exact string matching).
